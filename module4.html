Module 4

Inbound Logistics & QC

The bridge from the Supplier to the Warehouse. This module strictly separates Quantity Verification (Dock Receipt) from Quality Verification (Inspection), creating a "QC Firewall."

1. Advanced_Shipping_Notification (ASN)
[H] External

Trigger: The Supplier sends a digital packing slip. This updates the Purchase Order status to In-Transit in Module 2.

...
Triggers Physical Receipt (1:1)
2. Dock_Receipt (Quantity Check)
Business Event

Action: The truck arrives. Warehouse staff count the boxes.
Logic: Line 3 is a Short-Shipment (Ordered 26k, Received 25k).

...
Triggers Inspection (1:1)
3. Material_Inspection (Quality Check)
Business Event

Action: QC Lab tests samples.
Logic: Line 3 has 500 Failures. Only 24,500 passed.

...
Results In (1:1)
4. QC_Decision (Gateway)
Logic Gate

The "Firewall." The process splits based on inspection results.

PASS (24,500 Qty) Moves to Module 5: Material_Receipt (Available Stock).
FAIL (500 Qty) Moves to Module 5: Stock_Quarantine (Blocked Stock).

Logic & Data Definitions

The QC Firewall

We separate "Counting" (Dock Receipt) from "Inspecting" (Inspection). This ensures that we acknowledge receipt of goods legally (Dock) before declaring them fit for use (Inspection).

Feedback Loops

Dock_Receipt updates the Purchase_Order in M2 to "Received." This closes the loop on the "In-Transit" status.

Variance Handling

Short-Shipment: Detected at Dock (M4 Step 2).
Quality Failure: Detected at Lab (M4 Step 3).
Both result in a reduced "Available Qty" in M5.
