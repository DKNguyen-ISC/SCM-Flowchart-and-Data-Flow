MODULE 2: PROCUREMENT

Process Overview (v12.0)

This module handles the **external procurement** process. It is fed the "Net Demand" list from the M3 Balancing Hub.
Key Logic: The PIC (Purchasing-in-Charge) performs Sourcing (who to buy from) and Consolidation (bundling requests) to create the final `Purchase_Order` that is sent to the supplier.

1. Purchase_Request_Draft

This is the "Net Demand" list (the "Shopping List") generated by the **M3 Balancing Hub**. It serves as the starting point (To-Do List) for the PIC.

...
Input for Download (1:1)

2. Assign_Sourcing (Offline Excel)

This is an offline business event. The PIC downloads the "Draft" list and assigns a `SUPPLIER_ID` to each request based on price, lead time, and capacity.

Input: `Purchase_Request_Draft` (Net Demand List)

Refs: `Supplier_Information` (Master Data)

User: Chooses the `SUPPLIER_ID` for each line.

Output: Enriched file with `SUPPLIER_ID` selected.

Generates Upload (1:1)

3. Purchase_Request_Staging

The Planner's enriched file (now with supplier info) is uploaded to this staging table for validation.

...
Approved (1:1)

4. Purchase_Request_Final [H]

Once approved, the sourced requests are finalized. This is the "Approved-to-Buy" list, ready for PO consolidation.

...
Input for (N:1)

5. Manual_PO_Consolidation (Offline Excel)

The PIC reviews all "Approved-to-Buy" requests and **consolidates** them. For example, two separate requests for `300010406C` from the *same supplier* (`SUP-A`) are bundled onto a single PO (`PO-100`).

Input: `Purchase_Request_Final` (Approved-to-Buy List)

User: Groups lines by `SUPPLIER_ID` and assigns a `PO_ID`.

Output: A list of PO Lines, ready for upload.

Generates Upload (1:1)

6. PO_Upload_Staging

The consolidated POs are uploaded to a staging table for final validation (e.g., checking prices against supplier master).

...
Approved (1:1)

7. Purchase_Order [H]

This is the **final output of Module 2**: the official Purchase Order sent to the supplier. This PO record triggers two key downstream processes.

...
Output feeds M3 (Pegging) & triggers M4 (ASN)
