MODULE 6: INVENTORY CONTROL (AUDIT)

Process Overview (v14.0)

This module is **decoupled from daily operations** and handles the periodic **auditing** of our inventory.
Key Logic: An auditor performs a `Physical_Stock_Count` (an external event). The `Stock_Reconciliation` event compares this physical count to the `Stock_Data` (System Qty) to find a variance. This generates an `Inventory_Adjustment` which is posted to the `Stock_Ledger` (M5) to correct the master data.

1. Physical_Stock_Count

This process is triggered **externally** (e.g., weekly, monthly) by an auditor. They go to the warehouse bins with a scanner or clipboard and count the *physical* items. This data is captured as a "raw count" list.

3. Stock_Reconciliation (The "Compare" Event)

This is the **core logic of M6**. The system compares the **Physical Qty** (from the staging table) against the **System Qty** (from the `Stock_Data` master table in L1).
Exception Scenario: For `300010406C`, our system (after M5) shows a total of **30,500** units (24,500 + 6,000). The physical count only found **30,450**. This event calculates a **-50 unit variance**.
